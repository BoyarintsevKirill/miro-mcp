# Miro MCP Design Rules

Обязательные правила при работе с Miro через MCP. Цель: аккуратные, консистентные доски.

## Сетка и позиционирование

- **Шаг сетки:** 50px. Все координаты x, y кратны 50.
- **Отступ между элементами:** 30px минимум.
- **Отступ от края фрейма:** 40px.
- **Стикеры в ряд:** одинаковый y, шаг по x = ширина стикера + 30.
- **Стикеры в столбец:** одинаковый x, шаг по y = высота стикера + 30.

## Размеры

| Элемент | Ширина | Высота |
|---------|--------|--------|
| Sticky note (стандарт) | 200 | авто |
| Sticky note (компакт) | 150 | авто |
| Frame (секция) | 800 | 600 |
| Frame (большой) | 1200 | 800 |
| Shape (заголовок) | 300 | 60 |
| Card | 300 | авто |

## Цветовая палитра

### Sticky notes - семантические цвета
- **light_yellow** - основная информация, описания, факты
- **light_blue** - вопросы, гипотезы, идеи
- **light_green** - решения, результаты, плюсы
- **light_pink** - проблемы, боли, риски, минусы
- **yellow** - важные заметки, выводы
- **gray** - мета-информация, комментарии, примечания

### НЕ ИСПОЛЬЗОВАТЬ для стикеров
- red, dark_green, dark_blue, black, violet, orange - слишком агрессивные, создают визуальный шум

### Shapes и фреймы
- Фон фрейма: **#F5F5F5** (светло-серый) или без заливки
- Заголовки shapes: **#FFFFFF** фон, **#000000** или **#333333** бордер
- Коннекторы: **#333333**, ширина **2**

## Фреймы

- Каждый логический блок (CJM, Lean Canvas, Persona и т.д.) = отдельный фрейм
- Название фрейма = заголовок блока (title параметр)
- Между фреймами: минимум 100px отступ
- Фреймы выстраивать в сетку: максимум 3 в ряд по горизонтали

## Раскладка фреймов на доске

```
[Frame 1]  [Frame 2]  [Frame 3]
[Frame 4]  [Frame 5]  [Frame 6]
```

- Горизонтальный шаг между центрами фреймов: ширина фрейма + 150
- Вертикальный шаг: высота фрейма + 150

## Главное правило: один стикер = одна мысль

**НИКОГДА** не пихать несколько тем в один стикер. Каждый стикер содержит:
- Одну мысль, одну идею, один пункт
- Максимум 2-3 строки текста
- Если нужно перечисление -- каждый пункт на отдельном стикере

### Плохо:
```
ПРОБЛЕМА
1. Нет приоритизации
2. Нет адаптивного плана  
3. Зубрежка
```
(Три темы в одном стикере)

### Хорошо:
```
[Нет приоритизации]  [Нет адаптивного плана]  [Зубрежка перед сессией]
```
(Три стикера, каждый с одной мыслью)

### Исключения:
- Заголовки секций (можно длиннее -- это контекст, не мысль)
- Lean Canvas блоки (если стиль shapes -- один shape = один блок)

## Стикеры внутри фрейма

- Все стикеры одной категории = один цвет
- Раскладка: сетка или линия
- Максимум 5-6 стикеров в ряд
- Если стикеров > 6, переносить на новый ряд
- Текст: короткий, без длинных предложений. Ключевые слова, не абзацы

## Текст

- Заголовки секций: font_size **24** или **36**
- Подзаголовки: font_size **18**
- Основной текст на стикерах: оставлять дефолт
- Цвет текста заголовков: **#1A1A1A**

## Коннекторы

- Стиль: **normal** (сплошная линия) для основных связей
- Стиль: **dashed** для второстепенных/опциональных связей
- Цвет: **#333333**
- Ширина: **2**
- Подписи на коннекторах: только когда связь неочевидна

## Порядок создания

1. Сначала создать все фреймы с правильными позициями
2. Потом заполнять фреймы элементами (с parent_id!)
3. В конце добавлять коннекторы между элементами

## Чеклист перед публикацией

- [ ] Все стикеры одного типа одного цвета
- [ ] Все элементы выровнены по сетке
- [ ] Фреймы не перекрывают друг друга
- [ ] Между фреймами одинаковые отступы
- [ ] Нет "потерянных" элементов вне фреймов
- [ ] Заголовки фреймов заполнены
