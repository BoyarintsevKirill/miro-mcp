# miro-mcp

MCP сервер для работы с досками Miro. Позволяет Claude Code создавать, читать, редактировать и удалять элементы досок через Miro REST API.

20 инструментов: стикеры, фигуры, текст, фреймы, карточки, коннекторы, изображения, теги, массовые операции.

## Быстрый старт

```bash
git clone https://github.com/kirill-boyarintsev/miro-mcp.git
cd miro-mcp && npm install
claude mcp add miro -e MIRO_TOKEN=<ваш-токен> -- node /полный/путь/до/miro-mcp/index.js
```

Нет токена? Следуйте инструкции ниже.

## Как получить токен Miro API

Каждый создает свое приложение в своем аккаунте Miro. Это дает вам персональный API-токен с доступом только к вашим доскам. Никаких общих учетных данных, никакого сервера.

### Шаг 1: Откройте настройки разработчика

Зайдите на [miro.com/app/settings](https://miro.com/app/settings) и нажмите вкладку **"Your apps"** вверху.

Если вкладки "Your apps" нет, нужна Developer team. Miro предложит создать ее при первом создании приложения. Это бесплатно и занимает 30 секунд.

### Шаг 2: Создайте приложение

Нажмите **"+ Create new app"**. Заполните:

- **App Name**: любое название (например `Claude MCP`)
- **Developer team**: выберите вашу Dev team
- **Expire user authorization token**: НЕ ставьте галочку (чтобы токен не истекал)

Нажмите **"Create app"**.

### Шаг 3: Настройте разрешения

На странице настроек приложения прокрутите вниз до **Permissions** и поставьте галочки:

- `boards:read` -- чтение досок
- `boards:write` -- редактирование досок

Больше ничего включать не нужно.

### Шаг 4: Установите приложение и получите токен

Прокрутите вниз и нажмите **"Install app and get OAuth token"**.

Появится окно с двумя выпадающими списками -- **"Select an organization"** и **"Select a team"**. Это важно:

> Токен дает доступ только к доскам той команды, которую вы выберете.

Выберите команду, где лежат ваши доски, и нажмите **"Add"**.

Скопируйте токен. Он выглядит так: `eyJtaXJvLm9yaWdpbiI6...`

### Шаг 5: Подключите к Claude Code

Склонируйте репозиторий и установите зависимости:

```bash
git clone https://github.com/kirill-boyarintsev/miro-mcp.git
cd miro-mcp
npm install
```

Добавьте MCP сервер в Claude Code (из папки miro-mcp):

```bash
claude mcp add miro -e MIRO_TOKEN=<вставьте-ваш-токен> -- node "$(pwd)/index.js"
```

На Windows используйте полный путь вместо `$(pwd)`:

```bash
claude mcp add miro -e MIRO_TOKEN=<вставьте-ваш-токен> -- node C:\Users\you\miro-mcp\index.js
```

Перезапустите Claude Code. Готово!

## Подключение личных досок (не только Dev team)

По умолчанию токен работает только для той команды, которую вы выбрали при установке. Если ваши личные доски в другой команде/workspace, нужен дополнительный токен.

### Как получить токен для другой команды

1. Вернитесь в настройки приложения: [miro.com/app/settings](https://miro.com/app/settings) > **Your apps** > нажмите на ваше приложение
2. Прокрутите вниз и нажмите **"Install app and get OAuth token"** еще раз
3. На этот раз выберите **другую организацию/команду** из выпадающего списка
4. Нажмите **"Add"** и скопируйте новый токен

### Подключение нескольких команд к Claude Code

```bash
claude mcp add miro-work -e MIRO_TOKEN=<токен-рабочей-команды> -- node /path/to/miro-mcp/index.js
claude mcp add miro-personal -e MIRO_TOKEN=<токен-личной-команды> -- node /path/to/miro-mcp/index.js
```

Теперь Claude Code может работать с досками из обеих команд.

> **Совет**: чтобы увидеть все ваши команды, нажмите на название команды в левом верхнем углу настроек Miro. Появится выпадающий список со всеми командами.

## Использование

Запустите новую сессию Claude Code и говорите обычным языком:

```
> Покажи все мои доски в Miro
> Создай стикер "Привет мир" на доске <board-id>
> Построй Business Model Canvas с фигурами и стикерами
> Создай фрейм "Спринт 1" и добавь внутрь 5 карточек задач
> Соедини фигуры стрелками
```

ID доски можно найти в URL при открытии доски: `miro.com/app/board/<board-id>/`

## Доступные инструменты

| Категория | Инструменты |
|---|---|
| Доски | `list_boards`, `get_board`, `get_all_items`, `get_item` |
| Стикеры | `create_sticky_note`, `update_sticky_note`, `bulk_create_sticky_notes` |
| Фигуры | `create_shape`, `update_shape` |
| Текст | `create_text`, `update_text` |
| Фреймы | `create_frame`, `update_frame` |
| Карточки | `create_card` |
| Коннекторы | `create_connector` |
| Изображения | `create_image_from_url` |
| Теги | `create_tag`, `attach_tag` |
| Общее | `delete_item`, `move_item` |

### Поддерживаемые типы элементов

- **Стикеры** -- 16 цветов, кастомный размер, размещение внутри фреймов
- **Фигуры** -- прямоугольник, круг, треугольник, ромб, звезда, облако, стрелки, блок-схемы и другие
- **Текст** -- HTML-форматирование (`<b>`, `<i>`, `<a>`), кастомный размер шрифта и цвет
- **Фреймы** -- контейнеры для организации элементов
- **Карточки** -- карточки задач с заголовком, описанием и дедлайном
- **Коннекторы** -- линии/стрелки между элементами с подписями и стилями (сплошная, пунктирная, точечная)
- **Изображения** -- добавление по URL
- **Теги** -- цветные метки, прикрепляемые к элементам

### Не доступно через REST API

Комментарии, Kanban, Mind maps, Timelines, Таблицы, Голосования, Рисование.
